------------------------------------------------------------------------------
1. IS WISPR ALREADY IN THE WORKFLOW?
------------------------------------------------------------------------------
YES. Wispr Flow is already fully wired in as the speech-to-text layer for
Telegram voice notes. Nothing needs to be coded to "add" it — only the API
key is missing.

  - app/stt.py       : converts voice note -> 16kHz mono WAV (ffmpeg) and
                       calls the Wispr Flow REST API
                       (https://platform-api.wisprflow.ai/api/v1/dash/api)
  - app/bot.py       : handle_voice() downloads the voice/audio message,
                       calls stt.transcribe(), then feeds the transcript into
                       the same trip pipeline as a typed message
                       (handler: filters.VOICE | filters.AUDIO)
  - app/config.py    : reads wisprflow_api_key
  - Behavior         : if WISPRFLOW_API_KEY is empty, transcription raises and
                       the bot gracefully asks the user to type instead.
                       /health reports "wisprflow_configured": false.

  TO ENABLE: set in .env  ->  WISPRFLOW_API_KEY=fl-xxxxxxxx  then restart.

------------------------------------------------------------------------------
2. WHERE TO GET THE WISPR KEY
------------------------------------------------------------------------------
  - The Wispr Flow API is NOT public. Access is gated behind enterprise
    approval / partnership.
  - Request access: email enterprise@wisprflow.ai (or the partnership form).
  - Once approved: log in at https://platform.wisprflow.ai
    -> API Keys -> Create new key. Key (format "fl-...") is shown ONCE.
  - Docs: https://api-docs.wisprflow.ai/quickstart

------------------------------------------------------------------------------
3. TIMING & FREE VERSION (myth-busting)
------------------------------------------------------------------------------
  - NO fixed 24-hour turnaround. Approval is manual/human. Their docs say API
    partnerships are currently PAUSED and they "may not respond right away."
    Realistically: days to weeks, or no response.
  - NO free tier for the API. (The Wispr Flow desktop/mobile APP has consumer
    plans, but that is the end-user product, not this backend API.)
  - NOTE: the "24 hours" in .env is unrelated — TRIP_TTL_HOURS just controls
    how long a generated trip video stays live at its public link.
